종합소득세 환급 서비스, 1분

  • Role Product Designer, Front-End Developer
  • Duration2022.04 - 2022.05

1달 만에 오픈하게 된 배경

기존에 지엔터프라이즈에서 사업자를 대상으로 운영 중인 '비즈넵'에 5월 1일부터 신고가 가능한 종합소득세 신고 기능을 넣고 싶었지만, 매출이 높으면 간편장부/복식장부 같은 기반이 준비되어야 신고가 가능하기 때문에 준비 과정이 만만치 않았습니다. 그러나 '비즈넵'의 타겟 유저와는 다르지만 비교적 매출이 적은 프리랜서나 근로소득자를 대상으로는 '추계신고'(장부 작성 없이 소득 금액만을 기준으로 신고하는 기능)가 가능하기 때문에 비교적 적은 리소스가 들고, '삼쩜삼' 타겟과 비슷하기 때문에 시장성도 좋다고 생각했습니다.

'삼쩜삼'은 프리랜서를 대상으로 종합소득세 환급 서비스를 진행 중이고, '1분'은 근로소득 범위까지 포함하여 제공하는 서비스

근로소득 대상자를 고려하였지만, 연말정산을 진행한 경우가 많기 때문에 대부분의 사용자들이 '삼쩜삼'과 유사한 타겟일 것이라 생각하였습니다. '삼쩜삼'은 이미 시장성에 대한 검증이 되었기 때문에 '1분' 프로젝트를 시작하는 것에 있어서 구성원들 모두 긍정적이었습니다.

빠르게 TF 조직을 구성

3월 중순에 RnD팀에서 IR에 사용하기 위하여 근로소득자만을 대상으로 개발했던 것이기에 스펙이 낮았고, 갑자기 결정된 프로젝트여서 투입 가능한 스쿼드가 준비되어 있지 않았습니다. 그래서 벡엔드 3명, 프론트 1명, PO 1명, PD 1명을 모집하게 되었고, 비즈넵과는 다르게 B2C 대상자였기에 단독 서비스인 '1분'을 소개하기 위해 브랜딩팀과 협업하며 프로젝트를 진행했습니다.

빠른 완성을 위해 직접 개발

프로토타입은 따로 디자인을 적용하지 않은 상태로 만든 작업이라 모바일과 PC의 사용성에 맞게 디자인이 개편이 필요했습니다. 반응형에 맞게 수정하려면 break point(반응이 일어나는 지점)에 따라 모바일, 패드, PC로 3가지 형태의 디자인에 충분한 시간이 필요합니다. 또한 개발자와 의사소통의 비용도 만만치 않은 것을 고려하여, 직접 디자인과 개발을 진행하기로 했습니다. 이미 회사 내의 웹페이지를 반응형 웹으로 직접 개발했던 경험이 있었기 때문에 개발에 대한 부담은 적었고, 부족한 프론트 개발의 리소스를 해결하고 의사소통 비용을 줄여 빠르게 완성도를 높이기 위한 선택이었습니다.

초기 RnD팀의 프로토타입 화면으로 신고 플로우만 갖고 있는 상태

수정된 디자인

PC와 모바일의 반응형을 고려한 CTA 버튼

피츠의 법칙(Fitts' Law)에 따르면 CTA 버튼의 위치는 크고 가까울수록 사용자 편의성이 올라간다고 설명합니다. 세액 조회 시 PC에서는 금액에 대한 상세 내역을 안내한 뒤에 CTA 버튼을 하단에 두어 '맥락의 흐름 속에서' 전환율을 높이고, 모바일의 경우 CTA 버튼의 위치를 하단에 고정하여 추가 스크롤 없이 바로 인지할 수 있도록 하여 빠르게 '결제 및 신고'페이지로 이동할 수 있도록 하였습니다.

세액 조회 후, '환급금 받기' CTA 버튼 위치 수정

PC에서는 마우스를 오른쪽으로 사용하기 때문에 오른쪽에 결제 및 신고 기능을 하는 CTA 버튼 위치

추가로 영리 목적으로 운영하는 웹페이지의 경우 하단에 Footer(푸터)에 필수적인 정보를 고객들에게 알려주어야 의무가 있습니다. 그렇게 되면 모바일의 경우 CTA 버튼을 하단으로 고정할 경우 컨텐츠 몰입을 방해하기 때문에 결제 페이지 같은 경우에는 Footer를 제거하기로 하였고, Main(신고) 페이지 같은 경우에는 정보를 유지해야 해서 배경색을 옅게 수정하고 집중을 방해하는 Logo는 없애기로 했습니다.

PC와 메인(신고) 페이지의 Footer

애니메이션을 이용한 사용자의 몰입 향상

  • 컨텐츠의 몰입을 방해하지 않는 Header 영역

    사용자에게 시각적인 인지로 다른 영역인 것을 구분하는 방법은 배경색을 달리하는 것인데요. 1분도 세금 조회 전에 최상단 영역인 '1분 요약' 섹션을 구분해 주기 위해 배경색을 브랜드 컬러로 표현했습니다. 그리고 아래로 스크롤 하다 보면 세금 조회 섹션부터는 헤더의 색상을 컨텐츠와 같은 배경색으로 변경하여 넓은 화면에서 조금 더 사용자가 집중할 수 있도록 하였습니다.

    추가로 제휴처와 콜라보 하게 되어 로고를 같이 나타내야 했는데, 라이트 또는 블루같이 배경 테마에 따라 컬러를 수정하게 되면 제휴처의 로고 원색상을 훼손하게 되는 문제가 발생했습니다. 그래서 CSS 기능인 filter 속성 중에 brightness를 조절하여 배경이 어두운 블루 테마일 경우에 로고를 흰색으로 표현하여 로고의 원색상을 훼손하지 않고 표현할 수 있었습니다.

    '네이버 쇼핑'과 제휴했을 당시의 모습

  • 이탈률을 낮추기 위한 로딩 애니메이션

    사용자의 종합소득세를 조회하려면 홈택스(국세청)로 접근해야 합니다. 그러기 위해선 사용자의 홈택스 간편인증 과정을 지나면서 제대로 조회가 가능한지 검증을 하고, 검증이 통과되면 사용자의 소득 데이터를 계산하는 로직을 거쳐야 하기 때문에 사용자가 길게는 최대 1분 정도 기다려야 합니다. 조회하려면 필수적으로 기다려야하는 과정이기 때문에 이탈하지 않기 위해 시각적으로 움직이는 애니메이션이 필요했고, 기다리는 시간에 CS의 수를 줄이기 위한 방안으로 텍스트 애니메이션도 같이 추가하게 되었습니다.

  • 고객이 원하는 지점으로 빠르게 Flow 이동

    신고에 필요한 정보를 기입하는 과정이 하나의 페이지에서 이루어지기 때문에 다음 단계로 빠르게 이동하기 위해 '다음으로' 버튼을 옆에 추가하였고, 해당 인터렉션에 '스크롤 애니메이션'을 넣어 사용자 경험을 끊기지 않고 해당 지점으로 자연스럽게 이동할 수 있게 하였습니다.

  • 사용자의 감정을 극대화하는 축하 애니메이션

    '1분'을 이용하는 고객들은 모두 환급 금액을 기대하고 과정을 기다려온 만큼 마지막 계산 결과를 보여주는 타이밍이 중요하다고 생각했습니다. 그래서 마지막 단계에 환급 금액이 있을 경우 축하 애니메이션으로 사용자의 감정을 극대화하여 '1분' 서비스를 기억에 남을만한 인상을 남기게 하고, 되찾을 금액이 없는 고객에게는 세금 신고를 안내하는 화면을 보여줍니다.

신뢰도를 높이기 위해서 어떻게 해야할까?

사전신청 기간에 어떤 고객분이 '1분'이 피싱 사이트로 의심이 가서 세무소에서 진행하겠다고 문의를 남겨주어서 저희도 '1분'을 마주하게 될 신규 방문자들의 관점에서 생각해보게 되었습니다. 1분의 Main Page, 즉 신규 방문자들이 마주하는 페이지가 바로 간편인증을 하고 계산 결과를 바로 보여주는 페이지이기 때문에 빠르게 세액 조회를 해볼 수 있다는 장점은 있지만, 1분에 대한 의심으로 시도하지 않으려는 사용자도 적지 않을 것 같다는 생각이 들었습니다. 그래서 저희는 신뢰도를 높이기 위해 다양한 방법을 시도했습니다.

  • '1분 알아보기' page를 통해 '1분'에 대한 신뢰성 높이기

    메인 페이지에서 '1분 알아보기'로 접속한 고객들에게 1분을 운영하는 회사에 대한 소개를 먼저 하였습니다. '지엔터프라이즈'라는 회사 이름은 생소할 수 있지만, '네이버와 네이버파이낸셜이 동시 투자한 회사'라고 인지하는 것이 신뢰성 향상에 가장 좋을 것 같았습니다.

    그리고 고객분들이 많이 걱정하시는 보안에 대한 설명이 다음으로 오고, 메인 페이지에서 설명이 부재했던 1분의 '기능 소개'가 보충 설명되면서 다음으로 옵니다. 마지막은 들어오는 CS를 줄이고, 실제로 운영되고 있다는 안도감을 높이기 위해 '자주 묻는 질문'을 추가했습니다.

    저희는 '1분 알아보기' 페이지가 전환율에 도움이 되는지 데이터 애널리스트 분과 A/B 테스트를 진행했습니다. 4월 25일 '1분 알아보기' 페이지를 추가하고 나서 일주일 동안의 결과를 보니 1분에 방문한 고객(9,268명) 중 약 9%가 '1분 알아보기'를 방문했습니다. 방문 여부에 따른 간편 인증의 전환율을 보니 '1분 알아보기'를 방문한 고객이 유의미하게 전환율이 높다고 나왔습니다. 방문한 A 그룹의 전환율은 53.7%로 나왔고, 방문하지 않은 B 그룹은 전환율이 44.4% 나왔습니다. 그래서 '1분 알아보기' 페이지가 9.3% 더 높게 나와 인증 전환율에 도움이 된다는 사실을 알게 되었습니다.

  • Main 페이지의 최상단 영역 수정

    5월 10일까지 방문한 고객이 10만 명 이상이 되었을 때, 실제 조회 수가 4.8만이 되었습니다. 그중 결제한 환급 건수는 2640건이었고, 평균 환급액은 227,443원으로 지표가 나왔습니다. 이런 지표를 이용하여 신규 방문하는 고객들에게 서비스에 대한 신뢰도와 기댓값을 올리고자 했습니다.

    또한 앞서서 '1분 알아보기' 페이지에도 네이버와 네이버 파이낸셜이 있지만, 요약 페이지를 방문하는 유저가 9%로 추정되니 나머지 91%는 여전히 메인 페이지에 머무른다는 사실입니다. 그래서 메인 페이지의 최상단 영역에 '네이버' 투자사 로고를 넣기로 했고, 잘 보이는 Badge UI 형태로 타이틀 위에 두었습니다.

사용성 개선으로 전환율 높이기

정기 신고 기간인 1일부터 22일까지, 저희가 관심을 두고 있는 퍼널 중에 '간편 인증'을 하고 나서 '종합소득세 계산'까지 전환하는 퍼널이 평균 50%에서 점차 줄어들어 30% 구간까지 접어들고 있었습니다.

개선 전의 플로우는 사용자가 데이터 최대 8가지의 입력 단계를 지나야 하는 과정이어서 꽤나 긴 플로우를 지나야 '종합소득세 계산'까지 도달하게 됩니다. 단계를 확인하며 진행하는 플로우에서의 장점은 홈택스에서 조회된 데이터가 정확한지 사용자가 확인할 수 있지만, 단점으로는 마지막 단계까지 도달하는 데에 피로도가 상당할 수 있습니다.

홈택스에서 조회된 데이터가 있을 시, 해당 항목이 자동 추가됩니다

단계마다 실제로 추가로 입력하는 사용자가 많은지 조사를 해보니 '환급금 조회 시작'을 누른 사용자 수에서 '종합소득세 계산'까지 도달하는 경우가 44.7% 정도로 절반 이상이나 감소하였습니다. 하지만 '종합소득세 계산' 화면에서 '종합소득세 계산' 버튼을 누르는 경우가 99.6%에 달하였습니다.
