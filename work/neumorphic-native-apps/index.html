배경

스티브잡스가 집착하고 심혈을 기울이던 스큐어모피즘을 보면서 디자인을 하던 시절이 있었습니다. 그러나 잡스가 떠난 뒤, 플랫 디자인과 메터리얼 디자인 등의 미니멀리즘이 유행하면서 스큐어모피즘은 추억으로 남게 되었습니다. 인스타그램의 로고가 바뀐 날, 새로운 로고에 대해서 디자이너들이 안좋게 평가했던 기억도 이제는 추억이 되었네요.

지금은 디스플레이의 해상도가 엄청난 속도로 발전해서 예전의 그래픽 재질을 이미지로 넣기에는 퀄리티가 떨어져 보이기까지 합니다. 만약 스큐어모피즘을 적용하고자 한다면 디자이너들은 해상도에 맞추어서 이미지 사이즈를 배로 키워서 작업을 하게 되겠지요. 그러면 불필요하게 일의 작업량이 많아질 것입니다. 그것보다는 미니멀리즘이 유행하면서 UI 디자이너의 일이 점점 더 그래픽 쪽보다는 반복되는 UI를 체계화하는 작업에 집중하게되고 제품의 UX에 신경을 쓰면서 디자이너의 작업도 다른 형태로 변화하고 있습니다.

단순한 UI에 익숙해져 있을 즈음, 스큐어모피즘의 대안으로 새롭게 등장한 뉴모피즘을 보았습니다. 플랫디자인이 가지고 있는 단순함과 스큐어모피즘의 시각과 촉각을 아우르는 특성을 동시에 나타내고 있어서 디자이너들 사이에서는 이름부터 '-즘'을 붙여 하나의 트렌드로 불러지고 있었습니다. 그래서 개인작업을 통해 뉴모피즘의 장점과 단점을 파악하고 실제로 사용될 수 있는지 알아보고자 작업을 시작하였습니다.

Native Apps

느낀점

스케치앱에서 Shadow와 Inner Shadow를 사용하여 작업하였고, 웹페이지에서도 코드로 모양을 구현할 수 있어서 실제 앱에서도 동작할 수 있다고 보여집니다. Shadow에 의존한 그래픽 기법으로 복잡한 구성보다는 단순한 구성에 어울려 심플한 디자인에 사용하기 좋아보입니다. 실제 앱에는 다양한 직군과의 이해관계들이 얽혀있다보니 마케팅 요소로 2D 이미지가 들어가면 전체 밸런스가 무너질 것 같은 느낌이 듭니다. 개인적으로는 단순 네이티브 앱으로 사용하기에 좋다고 생각합니다.
